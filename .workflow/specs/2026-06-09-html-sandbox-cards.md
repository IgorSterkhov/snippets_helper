# HTML Sandbox Cards Spec

## Requirement

Allow Notes and Snippets to contain a single-file interactive HTML artifact,
such as an HTML presentation generated by Codex, and allow that artifact to be
shared through the existing public share-link flow.

## Chosen Direction

Implement **A. Sandbox HTML Card** for single-file HTML:

- users upload one `.html` / `.htm` file from the desktop Markdown toolbar;
- the editor stores a portable Markdown token:
  `![html:Title](https://.../v1/media/html/<token>)`;
- desktop preview renders the token as an HTML Card with a sandboxed iframe;
- public share pages render the token as the same sandboxed iframe card;
- raw inline HTML inside Markdown remains escaped and is not executed.

## Security Model

Uploaded HTML is treated as untrusted content.

- Public HTML is served from a tokenized URL and rendered only inside:
  `<iframe sandbox="allow-scripts">`.
- The iframe does not receive `allow-same-origin`, so its JavaScript runs in an
  opaque origin and cannot inspect the parent page, cookies, localStorage, or
  application DOM.
- HTML Card URLs are not arbitrary iframe URLs. The renderer accepts only the
  configured public HTML media base URL, or equivalent relative
  `/snippets-api/v1/media/html/<token>` paths, with a strict URL-safe token.
- The HTML response uses an explicit restrictive CSP intended for single-file
  artifacts: `sandbox allow-scripts`, `default-src 'none'`,
  `script-src 'unsafe-inline'`, `style-src 'unsafe-inline'`,
  `connect-src 'none'`, `frame-src 'none'`, `worker-src 'none'`,
  `object-src 'none'`, `base-uri 'none'`, `form-action 'none'`, and
  `X-Content-Type-Options: nosniff`.
- Public share pages also send a parent CSP with `frame-src 'self'`, so shared
  Markdown cannot embed arbitrary third-party iframes.
- Upload validation accepts UTF-8 HTML only, rejects empty files, path-like
  file names, non-HTML extensions, and files over the user's existing media
  upload limit.

## Storage

Reuse existing media storage and quota:

- create a `media_assets` row;
- create one `media_asset_variants` row with `variant='html'`,
  `mime_type='text/html; charset=utf-8'`, `width=0`, `height=0`;
- store the HTML file on disk under the existing media root with a tokenized
  `.html` filename;
- quota usage continues to sum stored media variant bytes.

No database migration is required.

## Desktop UX

Add an `HTML` toolbar button next to the Image button in Markdown editors used
by Notes and Snippets.

Modal behavior:

- choose `.html` / `.htm`;
- upload with the same progress channel as media uploads;
- show sandbox preview before insertion;
- insert the Markdown token on confirm;
- if the modal is closed before insertion, delete the created media asset.

Preview behavior:

- Markdown image tokens whose alt starts with `html:` render as an HTML Card;
- the card contains a compact header with title and an Open button;
- the iframe is sandboxed and visually constrained.

## Public Share UX

Existing note/snippet share pages render `![html:...](...)` tokens as HTML Cards
with sandboxed iframes. Plain raw HTML remains escaped text.

## Out Of Scope

- zip/project bundle upload;
- editing HTML inside the app;
- mobile HTML upload UI;
- pretty root `/snippets-html/...` URLs that require nginx changes.
